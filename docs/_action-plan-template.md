# ACTION PLAN TEMPLATE

This is the template of the action plan to be used for work planning in the project. It can used at differente levels, for large scope work or for specific enhancements, features, refactoring projects etc. The requirements for the action plan are the same.

- The plan is always composed as a chronological sequence of action items grouped into phases. The good plan contains up to 10 phases and each phase contains up to 10 action items. Phases describe the general purpose and the scope of work in the concise (but containing enough substance) form. Action items describe the work elements in details, preferably indicating what files to be modified. For convenience of status tracking each phase and each item shall include checkbox.
- When the plan changes, all the changes shall be documented prior to their implementation, so it must be accurate and reflect the current state of affairs.
- The ACTION PLAN shall not contain specs, requirements or architectural decisions (they should be a documented in a separate document or separate section of a bigger document that contain action plan). The action items can contain list of files that are affected by that item.
- The format/style of the action plan - the simplest possible, no markdown, no decorations, follow strictily the template below:
